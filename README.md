Чему я научился:
Сложное разрешение конфликтов слияния
— На практике отработал разрешение конфликтов в сложных архитектурных изменениях, когда несколько веток одновременно модифицируют одни и те же файлы.
— Узнал, как создавать гибридные решения, интегрируя изменения из разных веток.

Поиск регрессий с помощью git bisect
— Освоил технику бинарного поиска для точного определения коммита, в котором был внесён баг.
— Написал автоматический скрипт-тест для интеграции с git bisect run.

Работа с Git Submodules
— Научился добавлять, обновлять и управлять версиями подмодулей в проекте.
— Понял, как поддерживать сложные зависимости между репозиториями.

Создание кастомных Git Hooks
— Разработал сложные pre-commit и post-merge хуки для автоматической проверки кода и выполнения задач после слияния.
— Реализовал проверки на наличие debug-кода, сложность функций и потенциально медленные операции.

Продвинутые стратегии ветвления и слияния
— Создал сложную структуру веток: функциональные, рефакторинг, экспериментальные.
— Применил интерактивный rebase для очистки истории коммитов.

Работа с большими файлами через Git LFS
— Настроил отслеживание бинарных файлов (PSD, MP4, ZIP и др.) с помощью Git LFS.
— Узнал, как управлять большими активами без переполнения репозитория.

Использование Multiple Worktree
— Освоил параллельную работу в нескольких рабочих деревьях для изоляции задач (hotfix, эксперименты, документация).

Создание собственных Git-команд
— Разработал кастомную команду git sync-all для синхронизации всех веток с удалённым репозиторием
